# `understandable analyze`

Главный шаг сборки. Обходит проект, гоняет tree-sitter, собирает
типизированный граф и сохраняет его в `.understandable/graph.tar.zst`.

## Синопсис

```bash
understandable analyze [--full | --incremental] [--with-llm] [--review] \
                       [--auto-update | --no-auto-update] \
                       [--plan-only] [--scan-only]
```

## Флаги

### Режим пайплайна

| Флаг              | Эффект                                                                                  |
|-------------------|------------------------------------------------------------------------------------------|
| `--full`          | Принудительная полная пересборка, даже если граф уже есть.                              |
| `--incremental`   | Пропускает файлы, у которых blake3-хеш совпадает с сохранённым; пересобирает только срез. |
| `--plan-only`     | Dry-run: печатает JSON change plan в stdout, ничего не сохраняет. Требует `--incremental`. |
| `--scan-only`     | Печатает project-scanner JSON и выходит. Полностью обходит анализатор.                  |

### LLM-обогащение

| Флаг                   | Эффект                                                                                |
|------------------------|---------------------------------------------------------------------------------------|
| `--with-llm`           | Per-file сводка/теги/сложность через настроенного LLM-провайдера.                     |
| `--llm-model <id>`     | Переопределить `llm.model`. По умолчанию: `claude-opus-4-7`.                          |
| `--llm-max-files <N>`  | Ограничение на число файлов для LLM. `0` = взять `llm.max_files` (по умолчанию 50).   |
| `--llm-concurrency <N>`| Файлов параллельно. Задаётся в YAML; по умолчанию 4.                                  |
| `--llm-temperature <f>`| Температура сэмплинга. Задаётся в YAML; по умолчанию 0.2.                             |

Когда `llm.provider == "host"`, флаг `--with-llm` делегируется
markdown-агентам IDE, и бинарник свой LLM-цикл пропускает.

### Хуки и фидбэк

| Флаг                | Эффект                                                                       |
|---------------------|-------------------------------------------------------------------------------|
| `--review`          | После analyze печатает расширенную одностроку (счётчики файлов, удаления).   |
| `--auto-update`     | Пишет `.understandable/auto-update.signal` для подсказки IDE-агентам.        |
| `--no-auto-update`  | Подавляет файл-сигнал, даже если YAML-дефолт его написал бы.                 |
| `--name <project>`  | Переопределить имя проекта (CLI > settings > имя директории).                |

## Модель конкурентности

`analyze --with-llm` использует `tokio::JoinSet`, ограниченный
`tokio::sync::Semaphore` ёмкостью `llm.concurrency` (по умолчанию 4).
Каждый файл — одна задача; семафор кепает число одновременных
запросов к LLM, чтобы репозиторий из 10k файлов не сжёг твой
токен-бюджет и не упёрся в rate limit провайдера.

Эмбеддинги (когда `embed_on_analyze: true`) используют тот же паттерн
с `embeddings.concurrency` (по умолчанию 2).

## Output cache

`analyze --with-llm` ключует кеш ответов по:

- `file_hash` — blake3 содержимого файла.
- `prompt_hash` — blake3 системного промпта + id модели.

Повторный прогон на неизменных файлах стоит **ноль токенов**. Кеш
лежит внутри `graph.tar.zst` рядом с самим графом.

:::tip
Сочетай output cache с prompt caching (автоматически включён для
системного промпта Anthropic) и Batch API (скидка 50 %, SLA 24 ч) —
см. секцию про экономию в [архитектуре](../architecture).
:::

## Каскад настроек

У каждого флага есть YAML-эквивалент. Порядок разрешения:

1. **CLI-флаг** (высший приоритет).
2. **`understandable.yaml`** в корне проекта.
3. **Встроенные дефолты** (низший приоритет).

Так что `--llm-max-files 25` всегда побеждает `llm.max_files: 50` в
YAML, который всегда побеждает встроенный дефолт 50.

## Примеры

### Первый analyze, без LLM

```bash
understandable analyze
```

Эвристический граф. Самый быстрый. Перезапускай после каждого
осмысленного изменения.

### Инкрементальная пересборка после коммита

```bash
understandable analyze --incremental --review
```

Эту комбинацию использует post-commit хук — переэкстрагируются только
файлы, у которых поменялся blake3, и печатается одностроковая сводка.

### LLM-обогащение с лимитом по бюджету

```bash
export ANTHROPIC_API_KEY=sk-ant-...
understandable analyze --with-llm --llm-max-files 25 --llm-concurrency 2
```

Кепает прогон 25 файлами и 2 одновременными запросами. Повторный
прогон на неизменных файлах переиспользует output cache.

### Plan-only превью (dry-run)

```bash
understandable analyze --incremental --plan-only | jq .
```

Печатает JSON change plan с `files_to_reanalyze`,
`rerun_architecture`, `rerun_tour`. Используется post-commit хуком,
чтобы решить, звать ли LLM вообще.
